# 🏔 Высота — Система управления студией воздушной гимнастики

> Веб-приложение с доменной моделью для управления студией воздушной гимнастики.  
> Разработано специально для тренеров: управление учениками, тренировками и постановкой номеров.

🔗 Демо: http://91.229.9.74/  
👉 Можно быстро зарегистрироваться и посмотреть функционал

---

## ✨ Возможности

- 👥 **Управление учениками**
  - Создание и редактирование профилей
  - Отслеживание прогресса

- 🎭 **Программы номеров**
  - Составление и структурирование выступлений
  - Организация элементов и последовательностей

- 📅 **Учет тренировок**
  - Логирование тренировочных сессий
  - Хранение структурированных записей

- 🔐 **Авторизация и роли**
  - Аутентификация через JWT
  - Система ролей:
    - Тренер
    - Администратор
    - Владелец

- 🧩 **База трюков (в разработке)**
  - Централизованная система элементов воздушной гимнастики

---

## 🧠 Архитектура

### Frontend

- **Vue 3**
- **Pinia** (управление состоянием)
- **Axios** (работа с API)

📐 Используется архитектура **Feature-Sliced Design (FSD)**:
- Четкое разделение по фичам и слоям
- Масштабируемая структура
- Удобство расширения и поддержки

---

### Backend

- **Flask**
- REST API
- JWT аутентификация
- Role-based access control

---

### База данных

- **PostgreSQL**
- Реляционная модель, адаптированная под домен

---

### DevOps и деплой

- Приложение полностью контейнеризировано (**Docker**)
- Сервисы:
  - Nginx (reverse proxy)
  - Gunicorn (WSGI сервер)
  - PostgreSQL
  - pgAdmin

---

## 🚀 Запуск проекта

### Локально через Docker

```bash
git clone https://github.com/roman-lem/Vysota-web-app.git
cd Vysota-web-app

docker-compose up --build
```

## 💡 Почему этот проект интересен

Это не просто CRUD-приложение.

Проект решает задачи конкретного домена (воздушная гимнастика) и моделирует реальные процессы:
- учет тренировок
- построение номеров
- управление студией через роли

Проект демонстрирует:
- масштабируемую frontend-архитектуру (FSD)
- fullstack интеграцию (Vue + Flask)
- production-подход (Docker)
- работу с доменной логикой
